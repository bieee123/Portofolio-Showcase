Reflected Cross-Site Scripting (XSS) in cx.apps.binus.ac.id

Security Researcher Cybersecurity Analyst

Cross-Site Scripting (XSS) is a common web application vulnerability that allows attackers to inject malicious scripts into webpages. This article discusses a reflected XSS vulnerability identified in cx.apps.binus.ac.id.

Vulnerability Overview

The vulnerability was discovered in the login endpoint of the cx.apps.binus.ac.id domain. It allows attackers to execute arbitrary JavaScript in the context of a user's browser, potentially leading to session hijacking, data theft, or phishing attacks.

Severity Level

The severity of this vulnerability is rated as High due to the potential impact on user security and data privacy.

Steps to Reproduce

  1. Navigate to the affected endpoint at https://cx.apps.binus.ac.id/login.
  2. Inject a malicious payload into the input field, such as: <script>alert('XSS')</script>.
  3. Observe the execution of the script in the browser.

Proof of Concept

Below is an example of a crafted URL exploiting the XSS vulnerability:

https://cx.apps.binus.ac.id/login?redirect=%3Cscript%3Ealert('XSS')%3C/script%3E

When the victim clicks this link, the script executes, demonstrating the vulnerability.

Proof of Concept Image

Recommended Fixes

To mitigate this issue, developers should:

  • Implement input validation to sanitize user inputs.
  • Use proper output encoding to prevent script execution.
  • Adopt Content Security Policies (CSP) to limit the execution of unauthorized scripts.

Conclusion

Reflected XSS vulnerabilities pose significant risks to web applications and their users. It is essential to address these vulnerabilities promptly to maintain secure digital environments.
